# Project Name: The Shoe Company

## Description
"The Shoe Company" E-Commerce Platform is an ongoing project that provides a versatile and engaging online shopping experience for a diverse range of shoes. The core features include a dynamic home page for users, an admin dashboard for efficient platform management, user authentication (login, logout, and registration), dedicated pages for contact us and about us, and more features currently in development. The project employs a robust MySQL database with a well-structured schema, adhering to naming conventions and normalization principles. The front end is implemented using HTML, CSS, JavaScript, and jQuery to ensure a responsive design, delivering a consistent user experience across various devices.

## Tasklists
- [x] **Database Design:** Crafted a well-structured database schema adhering to naming conventions and normalization principles.
- [x] **Data Population:** Populated the database with diverse and relevant shoe data for comprehensive testing and demonstration.
- [x] **Website Design:** Implemented visually appealing and marketable designs for The Shoe Company's user home page, admin dashboard, and additional pages.
- [x] **User Authentication:** Incorporated features for user login, logout, and registration, enhancing the security and personalization of the platform.
- [x] **Contact Us and About Us Pages:** Developed dedicated pages to provide users with information about the company and facilitate communication.
- [x] **Admin Dashboard:** Created a dashboard for administrators with specialized functionalities for managing the e-commerce platform.
- [x] **JavaScript and jQuery Functionalities:** Infused interactive features into the website to enhance user engagement across different pages.
- [x] **Responsive Design:** Optimized the layout for consistent display across various browsers and devices.

## Usage Instructions
1. Set up a MySQL database and import database file **"the_shoe_company.sql"**.
2. Clone this repository to your web server directory.
3. Configure the database connection details in **.env** file.
